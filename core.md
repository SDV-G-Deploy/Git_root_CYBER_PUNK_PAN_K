# Chain Lab - Core (Signal District)

## 1) Fantasy / premise
Chain Lab — это подпольный полигон техно-района, где оператор одним точным выстрелом запускает управляемый хаос и превращает короткий рискованный эксперимент в ценный технологический результат.

## 2) Роль Chain Lab в общей игре Signal District
- Что игрок делает в Chain Lab: за короткие забеги на арене строит цепные реакции, оптимизирует выстрел и выжимает максимум награды из ограниченного числа ходов.
- Зачем это нужно для Raid Grid/Ops Console: режим производит редкие тех-ресурсы и перформанс-сигналы, которые усиливают рейды и питают долгосрочные решения в Ops Console.

## 3) Core loop
Прицелься -> запусти один выстрел -> раскрой цепь -> зафиксируй награду -> реши: retry или выход в мета-слой.

## 4) Why fun
- Высокая плотность решений: один выстрел имеет реальную цену.
- Сильный визуальный payoff: цепь дает ощущение "я спланировал это".
- Быстрый цикл попыток: ошибка сразу превращается в следующий осмысленный заход.
- Понятная формула прогресса: хороший run заметно ускоряет развитие в общей игре.

## 5) Skill expression (что осваивает игрок)
- Геометрию выстрела: угол, дистанцию, порядок касаний.
- Чтение поля: предсказание, какие узлы включатся в цепь.
- Управление риском: когда играть в стабильный результат, а когда в глубокую цепь.
- Экономическую дисциплину: как конвертировать результат run в мета-выгоду.

## 6) Win/Lose conditions (для MVP)
- Win: ключевые узлы арены очищены в рамках лимита выстрелов, run завершается с наградой.
- Lose: выстрелы закончились до выполнения цели, run завершен без бонуса за очистку.

## 7) Economy linkage
- Награды из Chain Lab:
  - `credits` (базовая валюта за score),
  - `tech_parts` (материал для апгрейдов),
  - `tech_module_chance` (шанс на редкий модуль),
  - `performance_tags` (теги качества run для модификаторов/контрактов).
- Потребление в Ops Console:
  - `credits` и `tech_parts` тратятся на апгрейды агентов и системные улучшения.
  - `tech_module_chance` конвертируется в rolling-drop редких модулей для Raid Grid и Chain-модификаторов.
  - `performance_tags` используются как условия для авто-правил, приоритетов и бонусных задач.

## 8) UX-принципы честности
- Причинность цепей: каждое срабатывание имеет явный источник и читаемый порядок.
- Читаемая траектория: до выстрела игрок видит понятный прогноз пути.
- Быстрый retry: перезапуск в один шаг, без лишних экранов и задержек.

## 9) Метрики успеха MVP
- `TTFun`: <= 20 секунд до первого осмысленного выстрела и видимой реакции.
- `retry_rate`: 45-75% run-сессий включают хотя бы один retry.
- `avg_run_duration`: 60-180 секунд.
- `chain_depth_avg`: 2.5-4.0 по успешным run.
- `reward_claim_rate`: >= 95% сгенерированных наград подтверждаются/забираются.

## 10) Что НЕ делаем в MVP
- Несколько биомов, сложные генераторы арен и сюжетные сценарии.
- Экзотические типы узлов и многослойные статус-эффекты.
- Сложный live-ops, PVP/кооп и тяжелую социальную обвязку.
- Глубокую экономику с большим числом валют и крафтовых веток.

## 11) Post-MVP evolution
- 3 направления роста режима:
  1. Контент: новые арены, наборы узлов, модификаторы условий run.
  2. Системы: продвинутые синергии цепей, риск-мутаторы, вариативные цели раунда.
  3. Прогрессия: сезонные контракты и режимы мастерства с уникальными наградами.
- Мягкая интеграция в мета-слой Signal:
  - сохранять награды Chain Lab полезными, но не обязательными для базового прогресса;
  - вводить мета-связи поэтапно через лимитированные конверсии и прозрачные коэффициенты;
  - открывать новые мета-правила постепенно, чтобы не ломать темп коротких run.
